# Notes Feature

## Jacob Bennett's Insights

*Source: [Never forget anything – The engineer's over-engineered knowledge management system](https://medium.com/p/8874d89a4d70); article is paywalled*

**Problem**: Thousands of notes (shopping lists, meeting notes, book outlines) became a "junk drawer" not a library. Search became too difficult. Lost more ideas than remembered.

**Non-negotiables** (inferred):

| Improvement | Description |
|-------------|-------------|
| **Searchability** | Full-text search across all notes. Must never become "too difficult to search through." Filters, operators, search inside documents. |
| **Structure over chaos** | Templates, tags, linking. Organize by action/project, not random folders. Prevents junk-drawer collapse. |
| **Distill, don't hoard** | Clear capture rules (what qualifies to add). Skimmable summaries. Ruthless distillation. Short, reusable content > archives you never re-read. |

## Apply to Systems Notes

- **Robust search** (Meilisearch)
- **Templates** by note type (meeting, research, shopping, etc.)
- **Tags and links** for organization
- **Optional weekly review nudges**
- **Capture rules** guidance in onboarding
- Organize by action/project, not random folders
